# HTML
> 全称是 `Hypertext Markup Language(超文本标记语言)`
 
HTML用一系列预定义的标签，来告诉浏览器：“这里是一级标题”、“这里是一张图片”、“这里是一个链接”等内容

## 一、HTML文件的基本结构

```HTML
<!DOCTYPE html>
<html>
<head>
    <title>我的第一个网页</title>
</head>
<body>
    <h1>你好，世界！</h1>
    <p>这是我的第一个网页，我成功了！</p>
</body>
</html>
```

1. `<!DOCTYPE html>`：**文档类型声明**。这是告诉浏览器：“请用最新的HTML标准来渲染这个页面”
2. `<html>`：**根标签**。像一个大容器，所有其他标签都必须放在这个标签里面，它表示这是一个HTML文档。
3. `<head>`：**头部**。这里面放的是“给浏览器看的信息”，而不是在网页上直接显示出来的内容。比如页面的标题、字符编码等。
4. `<title>`：**标题**。浏览器标签上显示的文本
5. `<body>`：**身体**。所有在网页上能看到的内容都必须放在这个标签里面。

## 二、常用文本标签

### 1. **标题标签**：`<h1>这是一级标题</h1>`
和 *Markdown* 类似，它也有六级标题
> *注意*：按照层级顺序使用标题，不要跳级（比如在 `<h1>` 后面直接跟一个 `<h3>`）

### 2. **段落标签**：`<p>这是一个段落</p>`

更改文本样式：
```HTML
    <p>
        <strong>加粗</strong>、<em>倾斜</em>、<u>下划线</u>、<s>删除线</s> 
    </p>
```
### 3. **列表标签**：`<li>这是一个项目</li>`

- 无序列表 `<ul>` 
例如：
```HTML
    <p>我需要到学生服务中心购买的物品：</p>
    <ul>
        <li>酸奶</li>
        <li>泡面</li>
        <li>柚子</li>
    </ul>
```

- 有序列表 `<ol>`
例如：
```HTML  
    <p>如何观看<i>《罗小黑战记2》</i>？</p>
    <ol>
        <li>充值一个B站大会员</li>
        <li>点击搜索栏，搜索《罗小黑战记2》</li>
        <li>点击全屏，开始观影吧！</li>
    </ol>
```

- 嵌套列表：顾名思义，有序列表与无序列表可以互相套娃
```HTML
<ul>
    <li><p>第一个有序列表</p><ol>
        <li></li>
        <li></li>
        <li></li>        
    </ol></li>
    <li><p>第二个有序列表</p><ol>
        <li></li>
        <li></li>
        <li></li>
    </ol></li>
</ul>
```

## 三、HTML属性

- 基本语法：`<开始标签 属性名="属性值">`
- 每个HTML元素可以具有不同属性
- 属性名称不区分大小写，属性值对大小写敏感：
```HTML
<img src="LXH.jpg" alt="">
<img SRC="LXH.jpg" alt="">
<img src="LXH.JPG" alt="">
```
前两者相同，第三个与前两个不同

1. **链接标签**：`<a>`
基本语法结构：`<a href="URL">"用户端显示的文本内容"</a>` 
    - `href`属性：用来连接 **希望用户点击后跳转到的地方**，可以是绝对路径（URL），也可以是相对路径（网站的其他页面）
```HTML
    <p>
        你可以通过 <a href="https://luoxiaohei.huijiwiki.com/wiki/">罗小黑百科</a>来了解更多关于罗小黑的内容
    </p>
```
2. **图片标签**：`<img>`  

基本语法结构：`<img src="路径" alt="未加载出来时显示的文字" width="调整图片大小">`
   -  `src` 属性：用来连接**图片地址**，可以是绝对路径（URL），也可以是相对路径（本地文件）
   -  `alt` 属性：用来连接**未加载出来时显示的文字**
   -  `width` 和 `height` 属性：调整图片大小，一般只用一个
  
## 四、HTML区块——块元素与行内元素

1. `<div>` 标签
   - **定义**：`<div>` 是一个**块级元素**），通常用作其他元素的容器。
   - **特点**：浏览器通常会在 `<div>` 元素前后放置换行符（独占一行）。它没有特定的含义，主要用于通过 CSS 对网页进行布局和样式化。
   - **用途**：文档布局、分组内容。

2. `<span>` 标签
   - **定义**：`<span>` 是一个**行内元素**，通常用于组合文档中的行内元素。
   - **特点**：它不会在前后创建换行，只占据它包含的内容所需的宽度。
   - **用途**：用于对文本的一小部分进行样式设置（如颜色、字体）。  

3. 区别示例

```HTML
<!-- div 是块级元素，会独占一行 -->
<div>
    <h3>这是一个 div 区域</h3>
    <p>div 里的内容</p>
</div>

<div>
    <p>这是另一个 div 区域</p>
</div>

<!-- span 是行内元素，不会换行 -->
<p>
    这是一段文字，<span style="color: red; font-weight: bold">这里是 span 包裹的重点文字</span>，文字继续在同一行显示。
</p>
```

### 补充(询问 AI 后整合而成)

1. `<section>` 和 `<div>` 的区别：
简单来说,**`<section>` 是有意义的“章节”，而 `<div>` 是无意义的“盒子”**。

### 核心区别对比表

| 特性 | `<section>` | `<div>` |
| :--- | :--- | :--- |
| **语义** | **有明确语义**。定义文档中的一个**主题性内容分组**，通常带有标题。 | **无任何语义**。纯粹是一个通用的内容容器，不表达任何特定含义。 |
| **用途** | 用于划分页面中独立的、完整的内容区块，如文章的章节、标签页中的面板、网页的独立区域（如简介、新闻列表、联系信息）。 | 仅用于**样式布局**或**脚本操作**，当没有更合适的语义化标签时作为“最后的选择”。 |
| **类比** | 一本书中的 **“第几章”** 。 | 搬家时的 **“空纸箱”** 。你可以把任何东西装进去，但箱子本身不告诉你里面是什么。 |
| **是否应包含标题** | **强烈建议**包含一个标题（`<h2>`-`<h6>`），以明确该章节的主题。 | 不需要，也不应该。 |

---

### 实例解析

例如构建我的个人主页：
```html
<!-- 正确用法：使用语义化标签 -->
<body>
    <header> <!-- 明确的 1>
    </header>

    <main> <!-- 明确的语义：页面主体内容 -->
        <section> <!-- 明确的语义：一个独立章节“关于我” -->
            <h2>关于我</h2> <!-- section通常应有自己的标题 -->
            <p>我是舒天宇，一名AI专业的学生...</p>
        <!-- 内部为了布局，可以使用div -->
            <div class="avatar-container">
                <img src="avatar.jpg" alt="头像">
            </div>
        </section>

        <section> <!-- 明确的语义：另一个独立章节“我的项目” -->
            <h2>我的项目</h2>
            <p>这是我的第一个网页项目...</p>
        <!-- 项目列表本身没有特定语义，用div包装以便布局 -->
        <div class="project-gallery">
            <div class="project-item">项目一</div>
            <div class="project-item">项目二</div>
        </div>
        </section>
    </main>
</body>
```

### 总结

1.  **首选语义化标签**：在 `<section>`、`<article>`、`<nav>`、`<header>`、`<footer>`、`<aside>`、`<main>` 等**语义化标签**都不适用时，才使用 `<div>`。
2.  **不要为了包装而包装**：如果只是为了添加样式（如圆角、阴影），并且没有明确的主题性内容，就大胆用 `<div>`。
3.  **嵌套是常态**：`<section>` 内部完全可以包含多个 `<div>` 来辅助布局，反之亦然。

在构建网页骨架时，先按**内容主题**用 `<section>` 划分大区块，再在区块内部用 `<div>` 进行细微的样式布局。这样写出的代码既专业又清晰。

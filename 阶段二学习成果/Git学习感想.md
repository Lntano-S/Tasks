## 学习内容

### 初识Git

刚看到 git 这个词的时候，我就觉得它和 Github 多少沾点关系，不过也是第一次接触这个词。在我认真阅读了**廖雪峰的 Git 教程**后，我了解到 Git 是目前世界上最先进的分布式版本控制系统，能自动帮我们记录每次文件的改动，还可以让同伴协作编辑，查看每次的修改内容，这可要比使用一般办公软件的效率高得多。

### Git的使用
刚打开终端页面时，满屏的命令让我有些手足无措，即使是阅读了**廖雪峰的 Git 教程**之后，我还是有点懵懵的。为了提交自己的第一个项目，Fork 我们 Geek 的 Repo 进行文件提交，我自己摸索半天也没整明白。后来我自己在b站上寻找相关的教学视频【【大学生扫盲课】1 Git、GitHub 和 Gitee 完整讲解：从基础到进阶功能】https://www.bilibili.com/video/BV1G8CFYvEjt?vd_source=5429a543e938f144e8efddcbb439f4f3  

初步学习了 Git 当中的常用命令，并在询问朋友后，我在我的 VScode 中实现了我的学习感想提交。

### 小结
当我在我的 Git 学习笔记中有条理地列出提交项目的操作步骤，当我用表格整理出 Git 的常用命令，当我第一次用 VScode 便捷地提交项目到我的 Repo...这一个个小瞬间不断激发着我的成就感，我心里想：使用Markdown形式进行笔记记录多是一件美事啊！对于我这种喜欢秩序的人来说，简直是正中我的下怀，所以我的学习兴趣一直也比较高涨。很喜欢这种点滴收获的感觉，希望自己在下一阶段的考核当中能做得更好:smile:

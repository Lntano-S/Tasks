# 这是我的 HTML 字典(询问 AI 后整合而成)

### 一、语义化块级容器（✅ 优先使用）

这类标签有明确含义，能告诉浏览器和开发者“这里面装的是什么”

| 标签 | 含义与用途 | 常见属性 | 示例 |
| :--- | :--- | :--- | :--- |
| **`<header>`** | **页眉或章节头部**。通常包含网站Logo、导航、标题等。 | `class`, `id` | `<header><h1>我的网站</h1></header>` |
| **`<nav>`** | **导航区域**。用于放置一组主要的导航链接。 | `class`, `id` | `<nav><a href="/">首页</a></nav>` |
| **`<main>`** | **网页主内容区**。一个页面只用一次，内容是独一无二的。 | `class`, `id` | `<main><p>这里是文章正文...</p></main>` |
| **`<section>`** | **独立的主题章节**。通常有标题，用于对内容分块。 | `class`, `id` | `<section><h2>第一章</h2><p>...</p></section>` |
| **`<article>`** | **独立、完整的内容块**。如博客文章、新闻、评论，可独立分发。 | `class`, `id` | `<article><h2>文章标题</h2><p>...</p></article>` |
| **`<footer>`** | **页脚或章节尾部**。通常包含版权信息、联系方式等。 | `class`, `id` | `<footer><p>© 2024 版权所有</p></footer>` |
| **`<figure>`** | **独立的流内容**（常配图）。与 `<figcaption>`（图注）一起使用。 | `class` | `<figure><img src="pic.jpg"><figcaption>图片说明</figcaption></figure>` |
| **`<aside>`** | **侧边栏或附加内容**。与周围内容相关，但不是主体（如侧栏、引用）。 | `class`, `id` | `<aside><p>相关阅读...</p></aside>` |

### 二、通用容器（🔄 无语义，用于样式和布局）

当没有合适的语义化标签时使用，主要作用是为CSS或JavaScript提供“钩子”。

| 标签 | 含义与用途 | 常见属性 | 示例 |
| :--- | :--- | :--- | :--- |
| **`<div>`** | **块级通用容器**。默认独占一行，是最常用的布局和组织元素。 | `class`, `id`, `style` | `<div class="card"><p>这是一个卡片</p></div>` |
| **`<span>`** | **行内通用容器**。不换行，用于包裹一小段文字或行内元素以单独设置样式。 | `class`, `id`, `style` | `<p>这是一段<span class="highlight">高亮</span>文字。</p>` |

### 三、使用建议与结构示例

**核心原则**：**优先使用语义化标签（第一部分）来搭建骨架，再用通用`<div>`进行内部细节的样式布局。**

一个清晰的页面结构应该是这样的：
```html
<body>
  <header>网站页眉</header>
  <nav>主导航</nav>
  <main>
    <article>
      <header>文章头部</header>
      <section>第一节</section>
      <section>第二节</section>
      <footer>文章尾部</footer>
    </article>
    <aside>侧边栏</aside>
  </main>
  <footer>网站页脚</footer>
</body>
```

**总结一下**：
- **布局分块**：用 `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`。
- **样式包裹**：用 `<div>` 和 `<span>`，但要在语义化标签内部使用。
- **避免“div滥用”**：不要整个页面都是 `<div>`，这会让代码像一锅没有结构的粥。

希望这份总结能成为你手边的速查手册。在实际编码时，多问自己：“这块内容是什么？”，答案往往就是该用的标签。

### 补充(询问 AI 后整合而成)

1. `<section>` 和 `<div>` 的区别：
简单来说,**`<section>` 是有意义的“章节”，而 `<div>` 是无意义的“盒子”**。

### 核心区别对比表

| 特性 | `<section>` | `<div>` |
| :--- | :--- | :--- |
| **语义** | **有明确语义**。定义文档中的一个**主题性内容分组**，通常带有标题。 | **无任何语义**。纯粹是一个通用的内容容器，不表达任何特定含义。 |
| **用途** | 用于划分页面中独立的、完整的内容区块，如文章的章节、标签页中的面板、网页的独立区域（如简介、新闻列表、联系信息）。 | 仅用于**样式布局**或**脚本操作**，当没有更合适的语义化标签时作为“最后的选择”。 |
| **类比** | 一本书中的 **“第几章”** 。 | 搬家时的 **“空纸箱”** 。你可以把任何东西装进去，但箱子本身不告诉你里面是什么。 |
| **是否应包含标题** | **强烈建议**包含一个标题（`<h2>`-`<h6>`），以明确该章节的主题。 | 不需要，也不应该。 |

### 实例解析

例如构建我的个人主页：
```html
<!-- 正确用法：使用语义化标签 -->
<body>
    <header> <!-- 明确的 1>
    </header>

    <main> <!-- 明确的语义：页面主体内容 -->
        <section> <!-- 明确的语义：一个独立章节“关于我” -->
            <h2>关于我</h2> <!-- section通常应有自己的标题 -->
            <p>我是舒天宇，一名AI专业的学生...</p>
        <!-- 内部为了布局，可以使用div -->
            <div class="avatar-container">
                <img src="avatar.jpg" alt="头像">
            </div>
        </section>

        <section> <!-- 明确的语义：另一个独立章节“我的项目” -->
            <h2>我的项目</h2>
            <p>这是我的第一个网页项目...</p>
        <!-- 项目列表本身没有特定语义，用div包装以便布局 -->
        <div class="project-gallery">
            <div class="project-item">项目一</div>
            <div class="project-item">项目二</div>
        </div>
        </section>
    </main>
</body>
```

### 总结

1.  **首选语义化标签**：在 `<section>`、`<article>`、`<nav>`、`<header>`、`<footer>`、`<aside>`、`<main>` 等**语义化标签**都不适用时，才使用 `<div>`。
2.  **不要为了包装而包装**：如果只是为了添加样式（如圆角、阴影），并且没有明确的主题性内容，就大胆用 `<div>`。
3.  **嵌套是常态**：`<section>` 内部完全可以包含多个 `<div>` 来辅助布局，反之亦然。

在构建网页骨架时，先按**内容主题**用 `<section>` 划分大区块，再在区块内部用 `<div>` 进行细微的样式布局。这样写出的代码既专业又清晰。
